# PDD-PROJECT (Agri Guard AI)

Agri Guard AI is a premium React + Vite progressive web application (PWA) designed to empower farmers with real-time AI-powered plant disease diagnoses, live meteorological forecasts, and agricultural mandi price intelligence. 

In addition to the core application, this repository contains a complete automated end-to-end (E2E) testing framework with 106 test cases and a Dynamic Application Security Testing (DAST) pipeline to evaluate vulnerability coverage.

---

## 📂 Repository Directory Structure

```
├── code files/                  # Core React + Vite progressive web application source code
├── automated_test/              # Dynamic Application Security Testing (DAST) runner & reports
├── Vulnerability Test Results/  # Executive summaries of vulnerability scanning results
├── E2E_Test_Report_*.xlsx       # Rich Excel sheets generated by the Selenium test runs
├── run_e2e_tests.py             # Selenium E2E test runner (106 test cases)
├── inspect_excel.py             # Utility to inspect generated Excel report contents
└── README.md                    # Main repository documentation
```

---

## 🚀 Key App Features

* 📱 **Plant Scan & Diagnose**: Upload leaf photos to receive instant organic & chemical treatment options, severity evaluations, and agronomical recommendations (powered by Google Gemini API).
* 🌦️ **Live Geolocation Weather**: Displays current and 7-day physical weather parameters (temp, wind, relative humidity, rain probability, uv index) fetched in real-time from the **Open-Meteo API** based on actual coordinates.
* 📊 **Indian Mandi Prices**: Real-time mandi rates and AI commodity price forecasts grounded through live **Google Search**.
* 🔌 **Hybrid Cloud Database (Supabase REST)**: Performs all data synchronization (profile, scan history, alerts) directly against a **Supabase database** in real-time if credentials are set, and falls back to browser `localStorage` if not set.

## 🛠️ Quick Start

### 1. Prerequisites
Ensure you have Node.js and Python installed.

### 2. Configure the App Codebase
Navigate into the `code files` directory:
```bash
cd "code files"
```
Install dependencies:
```bash
npm install
```
Copy `.env.example` to `.env` and fill in your Gemini API key (and optional Supabase credentials):
```env
VITE_GEMINI_API_KEY=your_gemini_api_key_here
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
```

### 3. Start the Web Server
Launch the development server:
```bash
npm run dev
```
Open **[http://localhost:5173](http://localhost:5173)** in your browser!

## 🧪 E2E Test Suite (106 Tests)

The repository includes a comprehensive E2E validation script in `run_e2e_tests.py` that utilizes Selenium WebDriver. It covers 106 individual test cases across:
1. **Landing & Auth**: Login flow, form validation, password visibility toggle, demo credentials.
2. **Crop Selection**: Commodity search, categorization, responsive layouts.
3. **Camera Capture**: Image upload triggers, low-light/blur handling simulation, camera overlay.
4. **Diagnosis & Actions**: Gemini LLM simulation, severity badges, report downloads, reset actions.
5. **Weather Dashboard**: Geolocation mapping, daily forecast charts, precipitation warnings.
6. **Mandi Prices**: Live search, commodity price trends, alert setup triggers.
7. **Alerts & Sync**: Critical alerts, read/unread filtering, database synchronization.
8. **Profile & Settings**: Local storage caching, language picking (English, Hindi, Tamil), dark mode.

### How to Run E2E Tests:
Make sure you have Chrome installed. Install the Python test dependencies:
```bash
pip install selenium openpyxl
```
Run the test runner from the root folder:
```bash
python run_e2e_tests.py
```
This script will automatically start the Vite dev server, run all 106 tests in a headless Chrome browser, terminate the server upon completion, and output a styled Excel spreadsheet named `E2E_Test_Report_AgriGuard_<timestamp>.xlsx` containing summaries, logs, and a clean breakdown of passed/failed tests.

## 🔒 Security Auditing (DAST)

The `automated_test` folder includes a Dynamic Application Security Testing script (`dast_runner.py`) designed to probe the application for vulnerabilities.

### How to Run Security Scans:
Run the script from the root folder:
```bash
python automated_test/dast_runner.py
```
This will generate JSON and Excel vulnerability summaries under `automated_test/` documenting safety assertions, input sanitation checks, and access control validation.
